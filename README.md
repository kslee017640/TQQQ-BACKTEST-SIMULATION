# V13.1 - 모드 분리 및 묶음 비교 개선

## 변경 내용
- 첫 화면에 `기존 백테스터` / `레버리지 묶음 비교` 모드 선택을 추가했습니다.
- 기존 백테스터에서 티커 검증 시 첫 데이터일을 함께 표시합니다.
- 빠른 기간 `전체` 선택 시 해당 티커의 첫 데이터일부터 시작일을 즉시 설정합니다.
- 단일기간 스윕과 다중기간 분할 스윕의 N 표기를 `EMA N일` 기준으로 명확히 바꿨습니다.
- 묶음 비교에서 거치식 초기자본과 분석기간을 기준으로 적립식 월 투자금을 자동 계산합니다.
- 묶음 비교에 `EMA/시그마 리밸런싱` 전략을 추가했습니다.
- 묶음 비교 결과를 전략별로 묶어 1배/2배/3배를 비교하기 쉽게 정리했습니다.

## 묶음 비교 전략
```text
1. 거치식
2. 적립식
3. 하락기준 적립식 리밸런싱
4. EMA/시그마 리밸런싱
```

## EMA/시그마 리밸런싱
- 매달 A만큼 정기매수하고 B는 현금으로 보관합니다.
- 가격이 EMA N일선보다 지정 시그마만큼 아래로 내려가면 보유 현금의 50%를 추가 매수합니다.
- 가격이 EMA N일선보다 지정 시그마만큼 위로 올라가면 보유 수량의 25%를 매도해 현금화합니다.
- 같은 방향의 신호가 매일 반복 발동하지 않도록 EMA 중심선 복귀 후 다시 발동되게 했습니다.
